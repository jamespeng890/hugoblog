Git是当前最流行的分布式版本控制系统，帮助开发者跟踪代码变化、协作开发和维护项目历史。

## 核心概念

- **仓库（Repository）**：存储项目所有文件和历史记录
- **提交（Commit）**：一次代码变化的快照，包含作者、时间和描述
- **分支（Branch）**：独立的开发线，允许并行工作
- **合并（Merge）**：将不同分支的修改整合到一起
- **远程仓库（Remote）**：存储在服务器上的仓库副本，用于团队协作

## 常用命令

1. `git init` - 初始化新仓库
2. `git add` - 添加文件到暂存区
3. `git commit` - 提交更改
4. `git push` - 推送到远程仓库
5. `git pull` - 从远程仓库拉取更新
6. `git branch` - 管理分支
7. `git merge` - 合并分支

## 工作流程

典型的Git工作流程包括：克隆仓库、创建分支、修改代码、提交更改、推送分支、创建拉取请求。

## 学习建议

从基础命令开始，逐步掌握分支管理、冲突解决和团队协作技巧。
